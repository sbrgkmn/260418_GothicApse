Gothic Apse

Purpose
Build an app that generates recursive Gothic apse forms using parametric rules derived from Jean Bony’s three-tier system.

Core Idea
- Generate apses from rules, not fixed geometry
- Maintain alignment between plan, elevation, and 3D
- Support recursion and variation within a coherent system

Inputs
- 5 apse types (each with plan, elevation, 3D)
- Each type should define:
  - proportions
  - tier relationships
  - basic geometric rules

What Codex Should Do
- Start simple, then expand
- Build one working apse generator first
- Then generalize to all 5 types
- Keep logic modular and readable
- Prefer reusable rules over hardcoded geometry

Minimal Architecture
- data/ → apse datasets
- lib/ → generation logic (recursion + params)
- components/ → UI
- types/ → schemas

Rules (Keep Loose)
- Use rule-based geometry, not image imitation
- Preserve plan ↔ elevation ↔ 3D relationships
- Recursion should be structured, not random
- Avoid overfitting to one precedent
- Keep system extendable

Interface (Simple)
- Select apse type
- Adjust parameters
- Control recursion depth
- Toggle plan / elevation / 3D

Success Criteria
- Generates coherent apses from rules
- Outputs stay consistent across representations
- System is understandable and adjustable
